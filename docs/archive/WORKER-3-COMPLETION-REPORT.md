## DMN File Structure (LEAN TIER-2 Format)

All generated DMN files follow this standardized structure:

### Required 5 Outputs
1. **resultado** - Decision result: "Prosseguir", "Bloquear", "Alertar", "Revisar"
2. **observacao** - Observation text (string)
3. **acaoRecomendada** - Recommended action (string)
4. **alertasConformidade** - Compliance alerts: "NENHUM", "DUP", "FREQ", "PRAZO", "DOC", "VALOR", "CRED", "CONTRATO"
5. **riscoDenial** - Denial risk: "BAIXO", "MEDIO", "ALTO", "CRITICO"

### Standard Components
- Valid XML with proper DMN namespace (https://www.omg.org/spec/DMN/20191111/MODEL/)
- Decision table with FIRST hit policy
- Minimum of 2 rules: approval rule + fallback rule
- DMNDI diagram section for visualization
- Portuguese language for all labels and content
- Comprehensive header comments with version, category, and perspective

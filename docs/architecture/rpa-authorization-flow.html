01 — Contexto
O problema que estamos resolvendo
Hoje, o processo de autorização junto à Unimed é manual. Um atendente acessa o portal, preenche dados e aguarda. Queremos automatizar isso.
🏥

Paciente na recepção

Paciente chega, precisa de autorização da operadora antes de ser atendido. Cada minuto importa.

Urgência
🤖

Robô executa no portal

RPA acessa o portal da Unimed, preenche o formulário de autorização e obtém o número automaticamente.

Automação
✅

Resultado em ~1 minuto

Autorização retorna ao sistema hospitalar sem intervenção humana. Atendente só clica em "confirmar".

Eficiência
🔌

Padrão para todos os robôs

03 — Detalhamento
Passo a passo técnico
O que acontece em cada etapa, com os dados que trafegam entre os sistemas.
🧑‍💼
T = 0s — Recepção
Atendente registra chegada e inicia processo
O sistema da recepção inicia um processo BPMN no CIB Seven. O processInstanceId é gerado e será a chave de correlação durante toda a execução.
⚙️
T = 1s — Worker MAEZO
Worker detecta Unimed e monta o payload
O RequestAuthorizationWorker identifica a operadora e prepara a chamada ao RPA com todos os dados necessários.
# Payload enviado ao RPA
{
  "processInstanceId": "abc-123-xyz", # chave de retorno
  "patientId": "PAT-9981",
  "payerId": "UNIMED_SP",
  "procedureCodes": ["10101012", "20101034"],
  "diagnosisCodes": ["J00", "J18.9"]
}
📨
T = 1s — HTTP fire-and-forget
Worker dispara o RPA e libera o processo BPMN
O worker faz um POST ao serviço RPA e retorna imediatamente (202 Accepted). O processo BPMN avança para o Message Catch Event onde fica aguardando — sem manter lock no Camunda.
POST http://rpa-unimed:8000/api/v1/authorize
→ 202 Accepted (RPA recebeu, processando)

# Worker completa a task imediatamente
# BPMN pausa em Message Catch Event
🤖
T = 1s → 55s — RPA executa
Robô acessa o portal da Unimed com Chrome headless
O container RPA usa Selenium + Chrome headless rodando com display virtual (Xvfb). Não há tela — tudo em memória no servidor.
# Dentro do container rpa-unimed (Docker Swarm)
1. Abre portal.unimed.com.br
2. Faz login com credenciais do hospital
3. Preenche dados do paciente + procedimentos
4. Submete solicitação de autorização
5. Aguarda resposta da Unimed (~30-50s)
6. Captura número de autorização
🔄
T = ~55s — Retorno ao BPMN
RPA correlaciona mensagem no CIB Seven
Usando o processInstanceId recebido no início, o RPA chama a API REST do CIB Seven para retomar o processo com o resultado.
POST http://cibseven:8080/engine-rest/message
{
  "messageName": "rpa_authorization_result",
  "processInstanceId": "abc-123-xyz",
  "processVariables": {
    "authorizationStatus": { value: "approved" },
    "authNumber": { value: "UNI-2026-98765" }
  }
}
✅
T = ~57s — Resultado
Processo retoma e resultado chega na recepção
O BPMN sai do Message Catch Event, processa o resultado e notifica a recepção. O atendente vê "AUTORIZADO — Nº UNI-2026-98765" na tela.

05 — Decisões de Arquitetura
Por que cada escolha foi feita
Comparação das alternativas avaliadas para cada decisão crítica.

Transporte Worker → RPA

Opção Lock Timeout Complexidade Resiliência Decisão
HTTP Síncrono (aguarda resposta) Alto risco Baixa Média Descartado
HTTP fire-and-forget + BPMN Message ✓ Escolhido Zero risco Média Alta Adotado
docker run por execução Alto risco Alta Baixa Descartado
Kafka + consumer RPA Zero risco Alta Muito alta Futuro

Solução adotada: HTTP fire-and-forget + BPMN Message Correlation

✓ Pontos fortes

Sem risco de lock timeout — processo BPMN fica parado no Message Catch Event, não no External Task
RPA pode demorar qualquer tempo — 1 minuto ou 30 minutos, o padrão não muda
Mesmo padrão para todos os robôs — Unimed, Bradesco, SulAmérica
Chrome já aquecido — sem overhead de startup por execução
2 replicas = 2 autorizações simultâneas, escalável pelo Swarm
Testável com mock HTTP — sem Docker em testes unitários
Auditoria completa no BPMN — cada etapa registrada com timestamp

— Pontos de atenção

RPA precisa estar rodando (serviço persistente) — mitigado pelo restart automático do Swarm
BPMN precisa de Message Catch Event — adiciona um elemento ao diagrama
RPA precisa conhecer a URL do CIB Seven para retornar o resultado
